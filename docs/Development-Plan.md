## 1. 开发总览

### 1.1 项目范围

| 模块 | 来源文档 | 阶段数 |
|------|----------|--------|
| TV 端 UI 改造 | [TVBOX-NEXT-UI-Redesign.md](./TVBOX-NEXT-UI-Redesign.md) | 10 阶段 |
| 手机端适配 | [Mobile-Adaptation.md](./Mobile-Adaptation.md) | 11 阶段 |
| **合计(去重后)** | - | **6 大阶段 / 20 子任务** |

### 1.2 开发原则

1. **基础设施先行**:SDK 升级、依赖添加、工具类创建优先
2. **TV 与手机并行**:UI 改造可并行开发,共享模块统一处理
3. **动画统一实现**:动画辅助类一次开发,TV/手机共用
4. **渐进式上线**:每阶段可独立验证,不依赖后续阶段
5. **风险后置**:海报 API、动态背景色等高风险项放最后

### 1.3 优先级定义

| 优先级 | 含义 | 上线要求 |
|--------|------|----------|
| P0 | 必须完成 | 阻塞发布 |
| P1 | 核心功能 | 建议随发布上线 |
| P2 | 增强体验 | 可延后到下个版本 |
| P3 | 锦上添花 | 按需排期 |

## 4. 里程碑与验收标准

### 4.1 里程碑定义

| 里程碑 | 完成任务 | 交付物 |
|--------|----------|--------|
| M1:基础就绪 | 阶段一全部 | 可编译的项目 + 设备识别 + 动画工具类 |
| M2:TV 版可用 | 阶段二全部 | TV 端 Netflix 风格界面 |
| M3:手机版可用 | 阶段三全部 | 手机端完整界面 |
| M4:动画上线 | 任务 4.1 | TV + 手机丝滑动画 |
| M5:一镜到底 | 任务 4.2 | 共享元素转场 |
| M6:海报增强 | 任务 5.1 | 海报库 API 集成 |
| M7:全量发布 | 阶段六全部 | 全设备适配 + 动态背景 |

### 4.2 发布版本规划

| 版本 | 包含里程碑 | 说明 |
|------|----------|------|
| v1.0-beta | M1 + M2 + M3 | TV + 手机基础版,无动画 |
| v1.0 | M4 | 正式版,含丝滑动画 |
| v1.1 | M5 | 增强版,一镜到底 |

## 5. 风险控制

### 5.1 关键风险点

| 风险 | 发生阶段 | 影响 | 应对 |
|------|----------|------|------|
| SDK 36 依赖冲突 | 阶段一 | 阻塞全部 | 锁定版本,逐步升级 |
| TV 焦点导航冲突 | 阶段二 | TV 不可用 | 保留原有导航逻辑 |
| 共享元素转场闪烁 | 阶段四 | 体验差 | 延迟过渡等待图片 |
| TMDB 访问不稳定 | 阶段五 | 无海报 | 降级豆瓣/原海报 |
| 老设备性能不足 | 阶段六 | 卡顿 | 提供精简模式 |

### 5.2 回滚策略

每个阶段独立提交,如出现问题:
1. 回滚到上一个里程碑版本
2. 修复后重新发布
3. 不影响其他阶段进度

### 5.3 并行开发协调

阶段二(TV)和阶段三(手机)并行开发时:
- **共享文件**:`util/anim/`、`util/poster/`、`res/anim/`、`res/xml/`、`res/transition/`
- **协调规则**:共享文件修改需通知两端
- **分支策略**:建议 TV 和手机分分支开发,合并时解决冲突
